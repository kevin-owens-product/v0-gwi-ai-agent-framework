- **Output Formats**: Types of deliverables the agent produces
- **Example Prompts**: Guided use cases for users

## Part 2: Workflow Architecture

### 2.1 What Are Workflows?

Workflows are multi-step automated processes that orchestrate agents, tools, and data transformations. They enable:

- **Automation**: Schedule recurring research tasks
- **Orchestration**: Chain multiple agents together
- **Data Flow**: Pass outputs between steps
- **Error Handling**: Retry and fallback logic

### 2.2 Workflow Step Types

The system supports **5 workflow step types**:

| Step Type | Purpose | Use Case |
|-----------|---------|----------|
| `agent` | Execute an agent | Run analysis or research |
| `tool` | Execute a GWI tool directly | Create audience, generate crosstab |
| `condition` | Conditional branching | If/then logic based on results |
| `transform` | Data transformation | Filter, map, aggregate data |
| `parallel` | Execute multiple steps simultaneously | Run multiple analyses at once |

### 2.3 Workflow Execution Architecture

```
                    ┌─────────────────────┐
                    │   Workflow Trigger   │
                    │ (Manual/Scheduled)   │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │   Step Executor     │
                    │ (Agent/Tool/etc.)   │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │   Context Manager   │
                    │ (Variables/Results) │
                    └──────────┬──────────┘
                               │
           ┌───────────────────┼───────────────────┐
           │                   │                   │
    ┌──────▼──────┐     ┌──────▼──────┐     ┌──────▼──────┐
    │  On Success  │     │  On Error   │     │   Retry     │
    │  (Next Step) │     │ (Fallback)  │     │  (Backoff)  │
    └─────────────┘     └─────────────┘     └─────────────┘
```

### 2.4 Example Workflow: Market Research Pipeline

```yaml
Workflow: "Product Launch Research"
Steps:
  1. Agent Step: "Audience Strategist"
     - Input: Target market, product category
     - Output: Audience definition, size estimate

  2. Tool Step: "create_audience"
     - Input: Audience criteria from Step 1
     - Output: Audience ID

  3. Tool Step: "generate_crosstab"
     - Input: Audience ID + competitor audiences
     - Output: Comparative data

  4. Agent Step: "Insights Generator"
     - Input: Crosstab data
     - Output: Key findings, recommendations

  5. Tool Step: "create_chart"
     - Input: Insights data
     - Output: Visualization

  6. Condition Step: "Check Significance"
     - If: Statistical significance > 95%
     - Then: Proceed to report
     - Else: Flag for manual review

  7. Agent Step: "Report Writer"
     - Input: All previous outputs
     - Output: Executive summary report
```

### 2.5 Workflow Data Features

| Feature | Description | Benefit |
|---------|-------------|---------|
| Variable Resolution | `{{step.output}}` syntax | Data flows between steps |
| Output Mapping | Map specific fields to variables | Selective data extraction |
| Retry Configuration | Max retries + exponential backoff | Fault tolerance |
| Parallel Execution | Run multiple steps simultaneously | Faster execution |
| Conditional Branching | If/then/else logic | Dynamic workflows |

### 3.3 Tool-Integrated Templates

Some templates have **built-in tool integration** that automatically executes GWI tools:

```typescript
// Example: International Market Comparison Template
{
  id: "international-market-comparison",
  toolIntegration: {
    suggestedTools: ["create_audience", "generate_crosstab", "create_chart"],
    preToolActions: [
      {
        toolName: "create_audience",
        parameters: {
          name: "[PRODUCT_CATEGORY] Consumers - [MARKET_1]",
          criteria: [{ dimension: "market", value: "[MARKET_1]" }]
        },
        storeAs: "audience1"
      },
      {
        toolName: "create_audience",
        parameters: {
          name: "[PRODUCT_CATEGORY] Consumers - [MARKET_2]",
          criteria: [{ dimension: "market", value: "[MARKET_2]" }]
        },
        storeAs: "audience2"
      }
    ],
    postToolActions: [
      {
        toolName: "generate_crosstab",
        parameters: {
          audiences: ["{{audience1.audienceId}}", "{{audience2.audienceId}}"],
          metrics: ["purchase_behavior", "media_consumption"]
        }
      }
    ]
  }
}
```

## Part 8: Implementation Guidelines

### 8.1 For New Agent Development

When creating new agents:

1. **Define Clear Capabilities** - List 5-7 specific actions
2. **Write Expert System Prompt** - Include domain expertise
3. **Create Example Prompts** - Provide 5+ use cases
4. **Specify Data Connections** - Which GWI datasets?
5. **Define Output Formats** - What deliverables?
6. **Add to Solution Area** - Organize by business function

### 8.2 For New Workflow Creation

When creating workflows:

1. **Map the Business Process** - What steps does the user do manually?
2. **Identify Agent Fit** - Which agents solve each step?
3. **Add Tool Steps** - Where is direct GWI access needed?
4. **Define Conditions** - What branches exist?
5. **Configure Error Handling** - What happens on failure?
6. **Test End-to-End** - Validate complete execution

### 8.3 For New Template Development

When creating templates:

1. **Identify Common Patterns** - What prompts are repeated?
2. **Extract Variables** - What changes each time?
3. **Add Tool Integration** - What tools should auto-execute?
4. **Write Clear Instructions** - Guide users to success
5. **Categorize Properly** - Research, Analysis, or Briefs?
6. **Add Tags** - Enable discovery

## Appendix C: Workflow Step Configuration

### Agent Step Config
```typescript
{
  agentId: string        // Agent to execute
  input: object          // Input parameters
  enableTools: boolean   // Allow tool usage
  allowedTools: string[] // Specific tools allowed
}
```

### Tool Step Config
```typescript
{
  toolName: string              // Tool to execute
  parameters: object            // Tool parameters
  outputMapping: {[key]: string} // Map outputs to variables
}
```

### Condition Step Config
```typescript
{
  expression: string     // Condition to evaluate
  trueStep: string       // Step ID if true
  falseStep: string      // Step ID if false
}
```

### Transform Step Config
```typescript
{
  inputMapping: {[key]: string}  // Source data paths
  transformations: [{
    field: string
    operation: "extract" | "map" | "filter" | "aggregate" | "format"
    config: object
  }]
  outputVariable: string         // Where to store result
}
```

### Parallel Step Config
```typescript
{
  steps: string[]        // Step IDs to run in parallel
  waitForAll: boolean    // Wait for all to complete
}
```
